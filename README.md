Enigma Hack Tech Sup Back
Бэкенд-часть решения для хакатона (команда Enigma). Репозиторий: https://github.com/Limekee/enigma_hack_tech_sup_back

📝 Описание проекта
Кейс: Создание автоматизированной системы для службы поддержки газовой службы.

Данный проект представляет собой API-ядро интеллектуальной системы технической поддержки. Основная цель решения — минимизировать время обработки обращений граждан и снизить нагрузку на операторов первой линии за счет автоматизации рутинных задач.

Ключевые возможности:

Автоматическая классификация входящих обращений (категоризация заявок).
Генерация черновиков ответов для операторов (или автоматические ответы на типовые вопросы).
Хранение истории обращений и контекста пользователя.
Интеграция с ML-моделями для анализа текста.
🏗 Основная архитектура
Проект построен на основе многослойной архитектуры (Layered Architecture), что обеспечивает разделение ответственности и легкое масштабирование.

Структура взаимодействия:
API Layer (Presentation): Принимает HTTP-запросы от фронтенда или внешних систем, валидирует входные данные (Pydantic схемы), передает управление сервисам.
Service Layer (Business Logic): Содержит основную логику приложения. Отвечает за маршрутизацию заявок, вызов классификаторов и формирование бизнес-ответов.
Data Access Layer (Repository): Взаимодействует с базой данных (CRUD операции), абстрагируя логику хранения данных.
ML Integration Layer: Отдельный модуль/микросервис, принимающий текст обращения и возвращающий предсказания (категория, срочность, предложенный ответ).
Схема данных:
Используется реляционная база данных для хранения пользователей, тикетов и логов диалогов. Взаимодействие происходит через ORM.

🚀 Запуск проекта
Требования
Установленный Docker и Docker Compose
(Или Python 3.11+ для локального запуска без Docker)
Запуск через Docker (Рекомендуемый)
Клонируйте репозиторий:
git clone https://github.com/Limekee/enigma_hack_tech_sup_back.gitcd enigma_hack_tech_sup_back
Создайте файл .env на основе примера .env.example и заполните переменные окружения:
env

DB_USER=postgres
DB_PASSWORD=your_password
DB_NAME=tech_support
SECRET_KEY=your_secret_key
Запустите контейнеры:
bash

docker-compose up --build
Сервер будет доступен по адресу http://localhost:8000.
Документация API (Swagger): http://localhost:8000/docs.

📌 Эндпоинты (Основные)
POST /api/v1/tickets/create — Создание нового обращения.
GET /api/v1/tickets/ — Получение списка обращений (с пагинацией и фильтрами).
POST /api/v1/classify — (Internal) Эндпоинт для классификации текста обращения.
GET /api/v1/stats — Получение статистики по обращениям.
